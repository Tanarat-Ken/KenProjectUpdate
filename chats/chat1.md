# เอกสาร 3 ใบ และวัตถุประสงค์ (ต้องสะท้อนในดีไซน์)
1) ใบเสนอราคา (QUOTATION) — เป้าหมาย: โน้มน้าวให้ลูกค้าตกลงจ้าง
   เน้น: ขอบเขตงาน, ราคาแยกรายการ, ยอดรวม, วันยืนราคา (valid until),
   เงื่อนไขการชำระเงินคร่าวๆ, สิ่งที่ลูกค้าจะได้รับ — โทน "ข้อเสนอ/ขาย" มองไปข้างหน้า

2) ใบส่งงาน (DELIVERY NOTE) — เป้าหมาย: ยืนยันว่าส่งมอบงานแล้ว และให้ลูกค้าตรวจรับ/ลงนาม
   เน้น: รายการงานที่ส่งมอบ (deliverables/checklist), วันที่ส่งมอบจริง,
   อ้างอิงใบเสนอราคา (Ref. QT), สถานะงาน, และ "ช่องลงนามตรวจรับงาน"
   (ผู้ส่งมอบ / ผู้ตรวจรับ + วันที่) — โทน "ส่งมอบ/ยืนยัน" ไม่ใช่เอกสารขาย
   ราคาไม่ใช่พระเอกของใบนี้ (อาจแสดงเป็นข้อมูลอ้างอิงเท่านั้น)

3) ใบวางบิล / ใบแจ้งหนี้ (INVOICE) — เป้าหมาย: ให้ลูกค้าจ่ายเงิน
   เน้น: "ยอดที่ต้องชำระ" ตัวใหญ่เด่นชัด, วันครบกำหนดชำระ (due date),
   ช่องทางชำระเงิน (เลขบัญชีธนาคาร / PromptPay), การหัก ณ ที่จ่าย 3%
   (แสดงเป็น ยอดก่อนหัก → หัก ณ ที่จ่าย 3% → ยอดสุทธิที่ต้องโอน),
   เงื่อนไขการชำระ, อ้างอิง QT/DN/PO — โทน "การเงิน/ทางการ"
   ต้องทำให้ "จ่ายเท่าไหร่ จ่ายยังไง" ชัดที่สุด

# ฟิลด์มาตรฐานเอกสารไทย (ใส่ให้ครบตามแต่ละใบ)
- ทุกใบ: ผู้ออกเอกสาร (ชื่อ/ตำแหน่ง/ติดต่อ), เลขที่เอกสาร, วันที่,
  ข้อมูลลูกค้า (ชื่อ/ผู้ติดต่อ/ที่อยู่/เลขประจำตัวผู้เสียภาษี),
  ตารางรายการ (ลำดับ / รายละเอียด / จำนวน / ราคาต่อหน่วย / รวม), หมายเหตุ, ลายเซ็น
- ใบเสนอราคา: Subtotal/ยอดรวม, วันยืนราคา, เงื่อนไขการชำระเงิน, ขอบเขตงาน
- ใบส่งงาน: วันที่ส่งมอบจริง, อ้างอิงใบเสนอราคา, รายการ deliverables,
  ช่องลงนามตรวจรับงาน (ผู้ส่งมอบ / ผู้ตรวจรับ + วันที่)
- ใบวางบิล/ใบแจ้งหนี้: ยอดที่ต้องชำระ (เด่น), วันครบกำหนด,
  เลขบัญชีธนาคาร / PromptPay, หัก ณ ที่จ่าย 3% (ยอดก่อนหัก→หัก→สุทธิ),
  เงื่อนไขการชำระ, อ้างอิง QT/DN/PO

## 📐 Agent Office — โครงสร้างระบบ (Requirement Classification)

**1. แกนข้อมูล (Entities)**
- **Project (งาน)** = หน่วยกลาง — ลูกค้า, ชื่องาน, รายละเอียด/คอนเซป, มูลค่า, สถานะ, timeline, ไฟล์แนบ, เอกสาร, ประวัติแก้ไข, log กิจกรรม
- **Client (ลูกค้า)** — ชื่อ/ผู้ติดต่อ/ที่อยู่/เลขผู้เสียภาษี (ใช้ซ้ำข้ามงานได้)
- **Documents (เอกสาร)** — QT → DN → INV → RC ผูกกันเป็นลูกโซ่ในแต่ละ Project แต่ละใบมีสถานะของตัวเอง
- **Files (ไฟล์แนบ)** — เก็บได้หลายสกุล เน้น **.md / .html** มี viewer เรนเดอร์อ่านในระบบ (ใช้บรรยายคอนเซปงาน)
- **Revisions (การแก้ไข)** — ประวัติเวอร์ชันต่อเอกสาร/ต่อ Project ว่า "แก้เรื่องอะไร เมื่อไหร่ ใครแก้"
- **Settings/Config** — ข้อมูลผู้ออกเอกสาร (ชื่อ, ตำแหน่ง, อีเมล, เลขผู้เสียภาษี, เลขบัญชี, PromptPay, โลโก้) → ไหลขึ้นเอกสารอัตโนมัติ แก้ได้อิสระ

**2. Workflow / Pipeline (Step ง่าย ๆ)**
`งานใหม่ → เสนอราคา → ลูกค้าตกลง → กำลังทำ → ส่งมอบ → วางบิล → รับเงิน → ปิดงาน`
แต่ละ step มีตัวบอกว่า "บันทึกธุรกรรมแล้วหรือยัง" และงานอยู่ step ไหน

**3. Dashboard (มุมมองรวม)**
- การ์ดสรุป: **งานค้าง** (กำลังทำ) · **งานรอ** (รอลูกค้าตอบ/รอจ่าย) · **งานรีวิว** (ส่งมอบแล้วรอตรวจรับ) · **เงินรอเก็บ**
- ยอดเงิน: เสนอไป / รอเก็บ / เก็บแล้ว (เดือนนี้)
- กิจกรรมล่าสุด + ปุ่มลัด "สร้างงานใหม่ / สร้างเอกสาร"

**4. ผู้ใช้ & สิทธิ์ (Roles)**
- **คุณ (เจ้าของ)** — เข้าถึงทุกอย่าง + แก้ Config/การเงิน
- **แฟน (ผู้ช่วย)** — โหมดสร้างเอกสารแบบ wizard ง่าย ๆ เห็นเฉพาะสิ่งที่ต้องทำ ไม่ยุ่งกับ Config

**5. หน้าจอหลัก**
Dashboard · รายการงาน · หน้า Project (แท็บ: ภาพรวม+pipeline / เอกสาร / ไฟล์ .md.html / ประวัติแก้ไข) · ตัวสร้างเอกสาร (wizard) · ตั้งค่า/Config
